# API Reference

This document summarizes the `pallet-content` runtime interface.

## Config

### `Config`

- `type WeightInfo: WeightInfo`
- `type ItemIdNamespace: Get<u32>`
- `type MaxParents: Get<u32>`
- `type MaxLinks: Get<u32>`
- `type MaxMentions: Get<u32>`

The runtime must provide bounds for parent item references, linked item references,
mentioned accounts, and an item-id namespace in addition to the pallet weight
implementation.

## Extrinsics

All calls are signed extrinsics and use their corresponding `WeightInfo`
functions.

### `publish_item`

```rust
publish_item(
    origin: OriginFor<T>,
    nonce: Nonce,
    parents: Vec<ItemId>,
    flags: u8,
    links: Vec<ItemId>,
    mentions: Vec<AccountId>,
    ipfs_hash: IpfsHash,
) -> DispatchResult
```

Creates a new item and emits a `PublishItem` event followed by `PublishRevision`
with revision `0`.

Rejects when:

- Any unsupported flag bit is set (`InvalidFlags`)
- The derived `ItemId` already exists (`ItemAlreadyExists`)

### `publish_revision`

```rust
publish_revision(
    origin: OriginFor<T>,
    item_id: ItemId,
    links: Vec<ItemId>,
    mentions: Vec<AccountId>,
    ipfs_hash: IpfsHash,
) -> DispatchResult
```

Increments the stored revision counter for the item and emits `PublishRevision`.

Rejects when:

- Item not found (`ItemNotFound`)
- Caller is not the owner (`WrongAccount`)
- Item has been retracted (`ItemRetracted`)
- Item is not revisionable (`ItemNotRevisionable`)
- Revision ID overflow (`RevisionIdOverflow`)

### `retract_item`

```rust
retract_item(origin: OriginFor<T>, item_id: ItemId) -> DispatchResult
```

Marks item as retracted and emits `RetractItem`.

Rejects when:

- Item not found (`ItemNotFound`)
- Caller is not the owner (`WrongAccount`)
- Item already retracted (`ItemRetracted`)
- Item is not retractable (`ItemNotRetractable`)

### `set_not_revisionable`

```rust
set_not_revisionable(origin: OriginFor<T>, item_id: ItemId) -> DispatchResult
```

Clears the `REVISIONABLE` bit in item flags and emits `SetNotRevsionable`.

Rejects when:

- Item not found (`ItemNotFound`)
- Caller is not the owner (`WrongAccount`)
- Item is already not revisionable (`ItemNotRevisionable`)

### `set_not_retractable`

```rust
set_not_retractable(origin: OriginFor<T>, item_id: ItemId) -> DispatchResult
```

Clears the `RETRACTABLE` bit in item flags and emits `SetNotRetractable`.

Rejects when:

- Item not found (`ItemNotFound`)
- Caller is not the owner (`WrongAccount`)
- Item is already not retractable (`ItemNotRetractable`)

## Storage

### `ItemState`

`StorageMap<Blake2_128Concat, ItemId, Item<AccountId>, OptionQuery>`

Maps each item ID to the current canonical control state:

- `owner`
- `revision_id`
- `flags`

## Storage helper

### `item`

Getter generated by pallet macros:

```rust
pub fn item(item_id: ItemId) -> Option<Item<T::AccountId>>
```

## Publicly visible types

- `pub struct Nonce([u8; 32])`
- `pub struct ItemId(pub [u8; 32])`
- `pub struct IpfsHash(pub [u8; 32])`
- `pub struct Item<AccountId> { owner, revision_id, flags }`

## Event payloads

- `PublishItem { item_id, owner, parents, flags }`
- `PublishRevision { item_id, owner, revision_id, links, mentions, ipfs_hash }`
- `RetractItem { item_id, owner }`
- `SetNotRevsionable { item_id, owner }`
- `SetNotRetractable { item_id, owner }`

## Errors

- `ItemAlreadyExists`
- `ItemNotFound`
- `ItemRetracted`
- `ItemNotRevisionable`
- `ItemNotRetractable`
- `WrongAccount`
- `InvalidFlags`
- `RevisionIdOverflow`
